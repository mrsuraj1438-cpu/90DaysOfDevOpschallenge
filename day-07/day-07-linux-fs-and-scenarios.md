# Linux File System Hierarchy (the most important directories)

Let's understand where things live in Linux.

## Core Directories (Must Know)

### / (root)

The base of the entire file system. All other directories branch from here.

Command:

```bash id="g59u5r"
ls /
```

I would use this to get into the root of Linux file system.

---

### /home

Home directories of users. Contains personal directories for each user. Users can use their own home directory without requiring permissions.

Command:

```bash id="i2uwra"
ls /home
```

I would use this to list home directories of all users.

---

### /root

Home directory of super user (root user) separate from normal users.

Command:

```bash id="b6vtnr"
sudo ls /root
```

I would use this to store files that only root should access.

---

### /etc

Contains configuration files of services, applications and the system itself. Also contains configuration files of users and passwords.

Command:

```bash id="p6fj1f"
ls /etc
```

Important files:

#### /etc/environment

I would use this to add environment variables so they’re accessible globally.

Command:

```bash id="t9gb0q"
cat /etc/environment
```

#### /etc/adduser.conf

I would use this to configure defaults for new users like shell, directory or group.

Command:

```bash id="bj06sj"
cat /etc/adduser.conf
```

---

### /var/log

Contains log files generated by services, applications and operating system. These logs are critical for troubleshooting, monitoring and auditing.

Command:

```bash id="7f5e5d"
ls /var/log
```

Important logs:

#### auth.log

I would use this to check logs related to authentication.

Command:

```bash id="df1kjx"
sudo cat /var/log/auth.log
```

#### syslog

I would use this to view general system and service related logs.

Command:

```bash id="rff96f"
sudo tail -f /var/log/syslog
```

---

### /tmp

Temporary files created by applications, services, or users are stored here.

Command:

```bash id="lpl2kl"
ls /tmp
```

I would use this to store temporary files that I won't require later.

---

# Additional Directories (Good to Know)

### /bin

Symbolic link to /usr/bin.

Command:

```bash id="oaxp3v"
ls /bin
```

---

### /usr/bin

Contains the majority of user command binaries. Programs and utilities that regular users and administrators run in daily work.

Examples:

* ls
* mkdir
* mv

Command:

```bash id="c1d9tv"
ls /usr/bin | head
```

I would use this to look for commands.

---

### /opt

Reserved for installing optional third-party software that are not part of core system distribution.

Example:

* Google Chrome

Command:

```bash id="mdzy5l"
ls /opt
```

I would use this directory to see what external or commercial applications are installed.

# Scenario-Based Practice

## Scenario 1: Service Not Starting

A service called 'ssh' failed to start after a server reboot.

### Step 1

Command:

```bash id="ll3m2l"
systemctl status ssh
```

Why:

* Check if the service is running or failed or stopped.

---

### Step 2

Command:

```bash id="e8n8lg"
journalctl -u ssh -n 50
```

Why:

* If service is failed check logs.

---

### Step 3

Command:

```bash id="u4y0e6"
systemctl is-enabled ssh
```

Why:

* To check if service starts automatically on boot.

---

### Step 4

Command:

```bash id="e2pqy8"
systemctl list-units --type=service
```

Why:

* Verify available services and check if ssh service exists.

---

## Scenario 2: High CPU Usage

Your manager reports that the application server is slow. You SSH into the server. What commands would you run to identify which process is using high CPU?

### Step 1

Command:

```bash id="x3mnd4"
top
```

OR

```bash id="h6ltgo"
htop
```

Why:

* List all the running processes. Check for processes that are CPU intensive.

---

### Step 2

Command:

```bash id="pkjgjq"
ps aux --sort=-%cpu | head -10
```

Why:

* Sort the processes by CPU percentage. Note down PID of top processes.

---

### Step 3

Command:

```bash id="2mbek5"
sudo renice +10 -p PID
```

Why:

* Increases the nice value, lowering the process priority so it gets less CPU time.

---

### Step 4

Command:

```bash id="xfu0ok"
kill PID
```

Why:

* Kill CPU intensive processes if necessary.

---

## Scenario 4: File Permissions Issue

A script at /home/user/backup.sh is not executing.

When you run:

```bash id="m35v94"
./demo.sh
```

You get:

```text id="r5gn0s"
Permission denied
```

### Step 1

Command:

```bash id="nslzcl"
ls -l demo.sh
```

Why:

* Check current permissions of file.
* Look for: -rw-r--r-- (notice no 'x' = not executable)

---

### Step 2

Command:

```bash id="jjd8ws"
chmod +x demo.sh
```

Why:

* Add execute permission to file.

---

### Step 3

Command:

```bash id="lqf5ep"
ls -l demo.sh
```

Why:

* Verify it worked.
* Look for: -rwxr-xr-x (notice 'x' = executable)

---

### Step 4

Command:

```bash id="qv9s4v"
./demo.sh
```

Why:

* Run the script.
